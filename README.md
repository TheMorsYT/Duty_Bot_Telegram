# DutyBot (aiogram 3 + SQLite)

Телеграм-бот для керування чергуваннями в групі.

## Що вміє зараз

- Ролі: `owner`, `admin`, `moderator`, `student`
- Черга чергових **по `sort_order`** (номери в черзі), а не по випадковому індексу
- Призначення чергових тільки на сьогодні
- Відмітка відсутніх на сьогодні (через кнопки, toggle)
- Заміна чергового (по прізвищах)
- Запити на заміну (`/skip_duty`) з апрувом/відхиленням
- Повідомлення автору запиту про результат (схвалено/відхилено)
- Повідомлення в групу про заміну: хто замінений на кого
- Загальна статистика:
  - по людях
  - по датах (10 кнопок на сторінку, нові -> старі)
  - у деталях дати показує і чергових, і відсутніх
- Нагадування owner о 10:30, якщо на сьогодні не призначено чергових
- Автоскидання `status` дня на `normal` на початку нового дня

## Важливі правила

### 1) Часове обмеження 09:00–14:00
Усі дії, що змінюють чергування, доступні тільки з `09:00` до `14:00`:

- призначення чергових
- заміна чергового
- відмітка відсутніх
- `/skip_duty` і його апрув
- зміна статусу дня (remote/normal)

### 2) Тільки сьогоднішнє призначення
`/assign` працює тільки на сьогодні.

### 3) Група не засмічується командами
У групі працює тільки `/set_group`.
Інші команди — тільки в ЛС з ботом.

## Команди

### Загальні
- `/start`
- `/menu`
- `/help`
- `/stats`
- `/stats_all`
- `/students`
- `/cancel`

### Student
- `/skip_duty [причина]`

### Moderator
- `/staff_list`
- `/rm_mod <TG_ID>`
- `/absent_on [YYYY-MM-DD]` (перегляд)

### Admin
- усе з Moderator плюс:
- `/assign`
- `/replace_duty <Старе_прізвище> <Нове_прізвище> [YYYY-MM-DD]`
- `/set_group_id <CHAT_ID>`
- `/unset_group`
- `/group_info`
- `/add_student <N> <TG_ID> <Прізвище> <Ім'я> [username]`
- `/edit_student <TG_ID> [last=...] [first=...] [username=...] [n=...]`
- `/del_student <TG_ID>`
- `/add_mod <TG_ID>`
- `/sync_user <TG_ID>`
- `/sync_students`

### Owner
- усе з Admin плюс:
- `/add_admin <TG_ID>`
- `/rm_admin <TG_ID>`

## Кнопки меню

`/menu` показує inline-меню за роллю.

Для `owner/admin` є `⚙️ Адмін-панель`, де винесені основні дії кнопками:
- призначити сьогодні
- заміна чергового
- відсутні на дату
- загальна статистика
- ролі
- група/статус

## База даних

Файл SQLite:
- за замовчуванням: `duty_bot.db` поруч зі скриптом
- або через `DB_PATH`

Основні таблиці:
- `users`
- `queue`
- `duty_log`
- `absences`
- `settings`
- `action_requests`

## Запуск

1. Встановити залежності:

```bash
pip install -r requirements.txt
```

2. Задати змінні:

- `BOT_TOKEN`
- `OWNER_ID`
- `DB_PATH` (опційно)

3. Запуск:

```bash
python main.py
```

## Примітка

`main.py` і `main1.py` синхронізовані (однакова логіка).
