# 🔔 DutyBot — Telegram Bot for Group Duty Management

Асинхронний Telegram-бот на базі **Aiogram 3** та **Aiosqlite**, призначений для автоматизації процесу призначення чергових у студентській групі або колективі. Бот підтримує гнучку систему ролей, облік відсутніх, роботу з запитами на заміну чергування та автоматичний збір статистики.

---

## ✨ Основні можливості

*   **Циклічна черга:** Автоматичний або ручний вибір **двох чергових** на основі індексу черги та індивідуального порядку сортування (`sort_order`).
*   **Чотирирівнева система ролей:**
    *   `owner` (Власник) — повний контроль над ботом, призначення адміністраторів.
    *   `admin` (Адміністратор) — керування чергою, додавання/редагування/видалення студентів, прив'язка групи.
    *   `moderator` (Модератор) — можливість ініціювати призначення чергових та відмічати відсутніх через систему запитів.
    *   `student` (Студент) — перегляд особистої статистики, черги та надсилання запитів на заміну.
*   **Система погодження дій (Approval System):** Запити від модераторів (на призначення, відмітку відсутніх, зміну статусу дня) та від студентів (на заміну через `/skip_duty`) надсилаються адміністраторам або власнику у вигляді зручних Inline-кнопок для підтвердження чи відхилення.
*   **Контроль призначення:** Захист від повторного призначення на ту саму дату. Якщо адмін або модератор намагається змінити вже існуюче призначення, бот автоматично запитує причину зміни та надсилає запит на затвердження власнику (`owner`).
*   **Часові ліміти:** Призначення чергових на поточний день дозволено лише у визначений проміжок часу (з **09:00** до **14:00**), щоб уникнути хаотичних змін посеред дня.
*   **Облік форс-мажорів:**
    *   **Дистанційний режим (`remote`):** Заморожує автоматичне або ручне призначення чергових на поточну добу.
    *   **Список відсутніх:** Можливість інтерактивно відмітити людей, яких сьогодні немає — бот автоматично пропустить їх під час вибору чергових.
*   **Автоматична синхронізація:** Бот автоматично перевіряє та оновлює актуальні `@username` користувачів безпосередньо з чату групи під час виклику списків або призначення.
*   **Фоновий моніторинг:** Щоденний асинхронний цикл перевірки: якщо о **10:30** чергові на сьогодні ще не призначені, бот надсилає нагадування власнику в особисті повідомлення.
*   **Інтерактивна статистика:** Перегляд загального рейтингу чергувань за прізвищами або детального журналу за датами з пагінацією сторінок (по 10 записів).

## 🛠 Технологічний стек

*   **Python 3.10+**
*   **Aiogram v3.x** (фреймворк для роботи з Telegram Bots API)
*   **Aiosqlite** (асинхронна бібліотека для роботи з SQLite)
*   **Asyncio** (керування асинхронними завданнями та фоновими циклами нагадувань)

## 🚀 Встановлення та запуск

### 1. Клонування репозиторію
```bash
git clone [https://github.com/yourusername/duty-bot.git](https://github.com/yourusername/duty-bot.git)
cd duty-bot
```

### 2. Створення віртуального оточення та встановлення залежностей
```bash
python -m venv venv

# Активація для Linux/macOS:
source venv/bin/activate
# Активація для Windows:
venv\Scripts\activate

pip install aiogram aiosqlite
```

### 3. Налаштування змінних оточення (Environment Variables)
Бот зчитує конфігурацію із системних змінних оточення. Ви можете задати їх у терміналі перед запуском або налаштувати у вашому хостинг-середовищі (наприклад, Docker, Systemd тощо).

*   `BOT_TOKEN` — Токен вашого бота, отриманий від [@BotFather](https://t.me/BotFather).
*   `OWNER_ID` — Цифровий Telegram ID головного власника бота.
*   `DB_PATH` — *(Опціонально)* Шлях до файлу бази даних SQLite. За замовчуванням створюється файл `duty_bot.db` у папці з ботом.

### 4. Запуск бота
```bash
# Приклад запуску з передачею змінних у Linux/macOS
BOT_TOKEN="YOUR_TELEGRAM_BOT_TOKEN" OWNER_ID=123456789 python main.py
```

## 👥 Командне меню бота за ролями

Залежно від ролі користувача, бот автоматично налаштовує список доступних команд у меню Telegram (через `set_my_commands`).

### 🔹 Загальні команди (Доступні всім)
*   `/start` — Запуск/перезапуск бота, реєстрація користувача в базі та налаштування персонального меню команд.
*   `/help` — Динамічний список команд, адаптований під поточну роль користувача.
*   `/menu` — Виклик головного інтерактивного Inline-меню.
*   `/stats` — Перегляд особистої статистики (кількість чергувань та записи в журналі).
*   `/stats_all` — Загальний звіт чергувань (за рейтингом прізвищ або календарем дат).
*   `/students` — Перегляд повного списку студентів у черзі відповідно до їхнього порядку.
*   `/cancel` — Скасування поточної дії або скидання стану FSM-форми (наприклад, введення причини пропуску).

### 🔸 Роль: Студент (`student`)
*   `/skip_duty` — Надіслати запит адміністраторам на заміну чи пропуск чергування (якщо викликано без аргументів, бот очікує введення причини наступним повідомленням).

### 🔸 Роль: Модератор (`moderator`)
*   `/assign` — Запустити процес призначення чергових на сьогодні або на конкретну дату.
*   `/staff_list` — Переглянути список адміністративного складу групи (`owner`, `admin`, `moderator`).
*   `/rm_mod <TG_ID>` — Зняти з себе або іншого модератора права (доступно також для admin/owner).

### 🔸 Роль: Администратор (`admin`)
*   `/set_group` — **Виконується безпосередньо в груповому чаті** для прив'язки бота до конкретної групи.
*   `/set_group_id <CHAT_ID>` — Ручне встановлення або зміна ID робочої групи бота через приватні повідомлення.
*   `/unset_group` — Відв'язати поточну групу від бота.
*   `/group_info` — Показати ID поточної прив'язаної групи та статус дня (`normal`/`remote`).
*   `/add_student <N> <TG_ID> <Прізвище> <Ім'я> [username]` — Додати нового студента в чергу під номером `N`.
*   `/edit_student <TG_ID> [Параметри]` — Зміна даних студента. Параметри передаються у форматі `last=НовеПрізвище`, `first=Ім'я`, `username=@user`, `n=номер`.
*   `/del_student <TG_ID>` — Видалення студента з активної черги чергувань.
*   `/add_mod <TG_ID>` — Надати користувачу роль модератора.
*   `/sync_user <TG_ID>` — Примусово оновити `@username` конкретного користувача, отримавши дані з чату групи.
*   `/sync_students` — Масове фонове оновлення юзернеймів усіх зареєстрованих студентів через Telegram API.

### 🔸 Роль: Власник (`owner`)
*   Має доступ до всіх існуючих команд системи + ексклюзивне керування адміністраторами:
    *   `/add_admin <TG_ID>` — Призначити користувача адміністратором бота.
    *   `/rm_admin <TG_ID>` — Понизити адміністратора до ролі звичайного студента.

## 📊 Структура бази даних (SQLite)

Під час першого запуску бот автоматично створює базу даних та застосовує необхідні перевірки обмежень цілісності (`PRAGMA foreign_keys = ON`).

*   `users` — Облікові записи користувачів. Містить ID, ПІБ, роль, лічильник чергувань `duty_count`, прапорець наявності приватного чату з ботом `has_pm`, індекс сортування `sort_order` та статус перебування в черзі `in_queue`.
*   `queue` — Зберігає поточний глобальний індекс черги (`current_index`) для циклічного перебору студентів.
*   `duty_log` — Журнал призначення чергових із фіксазий дати та ID обох напарників.
*   `settings` — Конфігурація бота (ID прив'язаного групового чату та поточний статус дня `status`).
*   `absences` — Таблиця фіксації пропусків. Зберігає дату, ID відсутнього студента та ID того, хто його відмітив.
*   `action_requests` — Черга системних запитів від користувачів, які очікують на апрув (`pending`, `approved`, `rejected`).
